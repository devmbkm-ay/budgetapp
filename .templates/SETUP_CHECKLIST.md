# Bun Monorepo Setup Checklist

Use this checklist when creating a new Bun monorepo project from scratch.

## Phase 1: Project Structure Setup

- [ ] Create root directory with `package.json` containing `"workspaces": ["apps/*", "packages/*"]`
- [ ] Create `apps/api/`, `apps/web/`, `packages/database/`, `packages/ui/` directories
- [ ] Create `lib/` directory for shared utilities
- [ ] Create `scripts/` directory for utility scripts
- [ ] Create `.templates/` directory to store templates for future projects

## Phase 2: Configuration Files

### Root Level
- [ ] `package.json` — Define workspaces and root scripts
- [ ] `bunfig.toml` — Copy from `.templates/bunfig.toml.root`
- [ ] `bun.lock` — Will be auto-generated by `bun install`
- [ ] `tsconfig.json` — Root TypeScript config (references workspace configs)
- [ ] `.env` — Optional, for local development
- [ ] `.env.example` — Copy from `.templates/.env.example`
- [ ] `docker-compose.yml` — Copy from `.templates/docker-compose.yml.bun-monorepo`
- [ ] `Dockerfile` — Copy from `.templates/Dockerfile.bun-monorepo`
- [ ] `.dockerignore` — Include: `node_modules`, `.next`, `.turbo`, `dist`, `build`

### App Level (apps/api/, apps/web/)
- [ ] `package.json` — Define dependencies (NO lockfile, NO workspaces)
- [ ] `bunfig.toml` — Copy from `.templates/bunfig.toml.app`
- [ ] `tsconfig.json` — App-specific TypeScript config
- [ ] `src/` — Source directory for app code

### Package Level (packages/database/, packages/ui/)
- [ ] `package.json` — Define dependencies
- [ ] `bunfig.toml` — Copy from `.templates/bunfig.toml.app` and adjust paths
- [ ] `tsconfig.json` — Package TypeScript config
- [ ] `src/` — Source directory

## Phase 3: Initial Setup

```bash
# In root directory
bun install

# Verify monorepo structure
bun workspaces list

# Build all packages
bun run build

# Start development
bun run dev:full

# Test Docker build
docker compose build --no-cache
docker compose up -d
```

## Phase 4: Docker Verification

- [ ] `docker compose ps` — All services running (not restarting)
- [ ] `curl http://localhost:3001/health` — API responding
- [ ] `curl http://localhost:3000` — Web app responding
- [ ] Check logs: `docker compose logs -f api`
- [ ] Health checks passing: `docker inspect <container> | grep -A 10 "Health"`

## Phase 5: Important Files to Commit

```bash
✅ Commit these:
- package.json (root)
- bun.lock ← CRITICAL: This ensures reproducible builds
- apps/*/package.json
- packages/*/package.json
- bunfig.toml (all levels)
- Dockerfile
- docker-compose.yml
- .dockerignore
- tsconfig.json (all levels)

❌ DO NOT commit:
- node_modules/
- .next/
- dist/
- build/
- .env (use .env.example instead)
```

## Common Gotchas & Fixes

### Issue: Module not found in Docker
**Cause**: Nested node_modules have broken symlinks  
**Fix**: See Dockerfile `RUN rm -rf` section - remove nested node_modules and create symlinks

### Issue: Dependencies not installed
**Cause**: Using `--frozen-lockfile` without `bun.lock` in git  
**Fix**: Commit `bun.lock` to version control

### Issue: App can't start locally but Docker is fine
**Cause**: Different Bun versions  
**Fix**: Update local Bun: `bun upgrade`

### Issue: Database connection fails in Docker
**Cause**: API starts before DatabasePostgreSQL is ready  
**Fix**: Use `depends_on` with `service_healthy` condition in docker-compose

## Next Time: Just Copy & Paste

For your next Bun monorepo project:

1. Copy `.templates/Dockerfile.bun-monorepo` → `Dockerfile`
2. Copy `.templates/docker-compose.yml.bun-monorepo` → `docker-compose.yml`
3. Copy `.templates/bunfig.toml.root` → `bunfig.toml`
4. Copy `.templates/bunfig.toml.app` → `apps/api/bunfig.toml` (and adjust for other apps)
5. Copy `.templates/.env.example` → `.env`
6. Run `bun install` and you're ready to go!
